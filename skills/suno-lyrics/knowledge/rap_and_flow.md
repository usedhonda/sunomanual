# ラップとフロー — 韻・ケイデンス・日本語ラップ設計

## 1. フロー類型

### 基本ルール
- ラップの質は「韻の数」だけでは決まらない。**flow = リズム + phrasing + accent + rhyme** の組み合わせで決まる。
- まずは on-beat で骨格を作り、その後に off-beat, double-time, triplet を部分導入する。
- 4 bars ごとに flow のどれか一要素を変えると停滞しにくい。
- 日本語では stress をそのまま再現せず、**モーラ位置と母音列**で flow を設計する。

### 例

**On-beat 例（英語）**
```text
I built this line to land on four
No drift, no slip, just bars that score
```

**Off-beat 例（英語）**
```text
I came in late but bent the grid
Not off the track, just slid and lived
```

**日本語例**
```text
まだここから まわるギア
わざと半歩だけ ずらすビート
```

### 類型表

| 類型 | 定義 | 使いどころ | 日本語応用 |
|------|------|------------|-----------|
| on-beat | 強拍や拍頭に要語を置く基本形 | 初見で意味を通したいVerse、宣言、パンチライン | 名詞・動詞の核を拍頭へ。助詞は弱位置へ逃がす |
| off-beat | 拍頭から少しずらして引っ掛かりを作る | 焦燥、攻撃性、前のめり感 | 2-3モーラ目から意味語を入れ、語境界を曖昧化する |
| double-time | 体感テンポを保ったまま音節密度だけ倍速方向へ上げる | 畳みかけ、緊張、技術誇示 | 2-4 bars の burst に限定し、助詞削減で密度を作る |
| triplet | 二分系の空間に三連系を差し込む rolling flow | trap, drill, chantable hook, ad-lib | 3モーラ単位を並べやすい。意味が軽くなりやすいので区画限定が安全 |

## 2. 韻スキーム

### 基本ルール
- end rhyme だけでなく、**internal rhyme と multisyllabic rhyme** で質感が決まる。
- 韻は「語尾一致」より「音列の再出現」と捉える。
- 日本語では stress-based rhyme より、**母音列・モーラ列の再現**が強い。
- 4 lines 全てを同強度で固めず、2 bars ごとに強弱を付ける。

### 例

**AABB（英語）**
```text
I map the room and hold the line
I bend the truth but keep the time
You talk in sparks and borrowed light
I walk through dark and call it sight
```

**内部韻（英語）**
```text
Cold in the corner, I fold what I order
Mind in a mortar, I grind every quarter
```

**日本語の母音韻**
```text
きみがいた
ひびがみだれ
いきがした
しびれきった
```

### パターン表

| パターン | ルール | 英語例 | 日本語例 |
|---------|--------|--------|---------|
| AABB | 2行ずつ pair rhyme | line / time / light / sight | かわる / たどる / よる / のこる |
| ABAB | 交互 rhyme | pass / wire / glass / fire | ひかり / こえ / まどり / そのえ |
| 内部韻 | 行中で音列を回収 | cold / fold / mortar / quarter | まちのなかで / かぜがながれ |
| 複合韻 | 1語と複数語、複数語どうしで音列対応 | data river / day to deliver | しょうめい / きょうもいい / とうけい |
| 多音節韻 | 2音節以上の塊で韻を作る | chemical panic / memory static | とけいのはり / どけいのたび |

### 日本語ラップ用メモ
- まず母音列を抜く: `きみがいた -> i i a i a`
- 次に近い母音列を探す
- 最後に子音を整える
- 音読み語は長い韻を作りやすいが、連発すると説明臭くなる

## 3. 16小節の構成法

### 基本ルール
- 16 bars は 1 ブロックではなく、**4 x 4 bars** として設計する。
- 各 4-bar stanza で rhyme / cadence / density の最低1つを変える。
- 最初から最後まで高密度にしない。後半で burst を作る方が強い。
- Hook 接続前は情報量を少し引いて、覚えやすい着地を用意する。

### 例

**4 x 4 の基本配分**
```text
bars 1-4   : 世界観提示 / 読みやすさ優先
bars 5-8   : detail / internal rhyme 増量
bars 9-12  : escalation / double-time or triplet burst
bars 13-16 : punchline / 視点反転 / hook への橋渡し
```

### ペーシング表

| 区画 | 役割 | 密度の目安 | 注意点 |
|------|------|-----------|--------|
| bars 1-4 | world / thesis | 低〜中 | いきなり詰め込みすぎない |
| bars 5-8 | detail / complication | 中 | internal rhyme を1-2箇所追加 |
| bars 9-12 | escalation / technical display | 中〜高 | double-time, triplet はここに寄せる |
| bars 13-16 | land / pivot / bridge to hook | 中→低 | 意味を回収し、hook の言葉を立てる |

### 日本語応用
- bars 1-4: 8-10モーラ中心
- bars 5-8: 8-12モーラ
- bars 9-12: 10-14モーラ burst 可
- bars 13-16: 6-10モーラへ戻し、hook を受けやすくする

## 4. ワードプレイ

### 基本ルール
- ラップの言葉遊びは、**意味のズレ**と**音のズレ**を同時に使うと強い。
- ダブルミーニング、ホモフォン、パンチラインは 4-bar 単位で設計すると効く。
- 日本語では同音異義語、漢語/和語のレジスタ差、当て字感が武器になる。
- ダジャレ化を避けたいなら、行末だけでなく**行中**に仕込む。

### 例

**ダブルミーニング**
```text
I kept it light but carried weight
```

**ホモフォン**
```text
I need peace, not just a piece
```

**日本語パンチライン**
```text
しょうめいはある でもまだ照明のした
```

### 技法表

| 技法 | 何をするか | 効きやすい場所 | 日本語応用 |
|------|-----------|---------------|-----------|
| ダブルミーニング | 1つの句に2解釈を持たせる | line end, hook word, title | 証明 / 照明、解く / 説く など |
| ホモフォン | 音が近い語で意味を滑らせる | line internal, response line | ないと / night、せい / say |
| パンチライン | 直前の setup を最後の1行で反転 | 4行目、8行目、16行目 | 1-3 bars を伏線にして最後で返す |

## 5. ケイデンスとデリバリー

### 基本ルール
- cadence は「どのタイミングに、どの長さで、どの強さの syllable を置くか」の反復パターン。
- 休符も cadence の一部。詰めるだけでは groove が死ぬ。
- デリバリーは lyric の意味と合わせる。怒りなのに laid-back だと温度が割れる。
- 日本語では `っ`, `ん`, 長音, 語切りで attack 感を作る。

### 例

**laid-back 例**
```text
beat の後ろに少しもたれて入る
```

**percussive 例**
```text
子音の打点を立て、短く切る
```

### バリエーション表

| タイプ | 感触 | 向く内容 | 日本語での作り方 |
|-------|------|---------|----------------|
| conversational | 話し言葉に近い | 物語、観察、独白 | 句読点を多めにし、行長を揃えすぎない |
| percussive | 打点が強い | battle, flex, drill | 促音、撥音、短語の連結で切る |
| laid-back | 拍の後ろに少しもたれる | 冷静さ、余裕、皮肉 | 語尾を伸ばし、休符を明示する |
| urgent | 前のめりで詰める | 焦燥、confession、追跡 | 助詞を減らし、短い語を連打する |
| staccato triplet | 粒立ちの強い三連 | trap hook, chant | 3モーラ単位で刻む |
| slurred / elastic | 音節境界を滑らせる | 浮遊感、酩酊感 | 語境界をまたぐ配置で流す |

## 6. 日本語ラップ特化ルール

### 基本ルール
- 英語の stress grid をそのまま持ち込まない。日本語は**モーラ grid**で考える。
- 韻は「母音列の再出現」として設計する。
- 子音は厳密一致より、**言いやすさと流れ**を優先する。
- 音読み語は technical rhyme に強く、和語は情感に強い。両方を混ぜる。

### 例

**母音列アンカー**
```text
きみがいた  / ひびがみだれ
いきがした  / しびれきった
```

**音読み語クラスター**
```text
証明 / 共鳴 / 透明 / 革命
```

### 実務表

| 項目 | ルール | 目的 |
|------|--------|------|
| 母音列フロー | まず bar end の母音列を揃える | 聴感上の一体感 |
| モーラベース韻 | 2-4モーラ単位で反復 | 日本語らしい flow の安定 |
| 子音配置 | ラ行・ナ行・マ行で流し、破裂音は要所だけ | 言いやすさと punch の両立 |
| 漢語/和語ミックス | technical line は漢語、感情 line は和語寄り | 単調回避と意味の温度差 |
| 生成順序 | 意味下書き -> 母音整形 -> 子音調整 | 袋小路を避ける |

### GPT への落とし込み
- `flow_type`: on-beat / off-beat / double-time / triplet
- `cadence_density`: sparse / medium / dense
- `rhyme_mode`: end / internal / multisyllabic / block
- `bar_pacing`: 4x4 / 8+8 / free
- `wordplay_mode`: punchline / double meaning / homophone / none
- `jp_rhyme_strategy`: vowel-chain / on-yomi cluster / mixed
